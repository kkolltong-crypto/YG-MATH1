# 과세특(세부능력 및 특기사항) 자동 생성 프로그램 — 요구사항 명세서

## 1. 목적

교과별 평가계획서를 입력받아 수행평가 결과를 분석하고, 학생부 "세부능력 및 특기사항"(과세특) 문구를 자동으로 서술해주는 프로그램.

## 2. 전체 처리 흐름

1. 각 교과의 평가계획서를 분석한다.
2. 평가 횟수와 평가 내용(수행평가 종류, 배점 등)을 분석한다.
3. 각 평가별 채점 기준을 분석한다. 예: 모두 이해 2점, 부분 이해 1점, 이해하지 못함 0점.
4. 반별로 작성하며, 학생 이름 대신 학번만 표시한다.
5. 한 반의 전체 내용을 한 장에 자연스럽게 통합 서술한다. 문장은 학생 생활기록부 형식에 맞추어 "~함", "~임"으로 끝맺는다.

## 3. 수행평가별 작성 단계 (5단계, 화면 순서 준수)

수행평가마다 아래 순서를 그대로 따른다.

| 단계 | 명칭 | 내용 |
|---|---|---|
| STEP 1 | 성취기준 선택 | 단원(교과서 출판사·학년 목차)별로 정리된 성취기준 중 평가할 기준을 선택. 선택 시 성취기준 문장 전체와 성취수준(A·C·E)이 함께 적용됨 |
| STEP 2 | 평가요소 확인·수정 | 성취기준에서 자동 추출된 평가요소 후보를 "지식이해 + 과정기능 + ~하기" 형태로 다듬고, 이번 문항에서 실제로 평가할 요소만 체크 |
| STEP 3 | 문항 초안 생성 | 선택된 성취기준·평가요소를 바탕으로 문항 초안 생성 |
| STEP 4 | 채점기준표(부분점수) | 문항별 부분점수 포함 채점기준표 생성 |
| STEP 5 | 교사 검토·확정 | 교사가 최종 검토 후 확정 |

### STEP 1 상세 — 성취기준 선택
- 교과별 성취기준을 직접 선택할 수 있는 방식, 또는 평가계획서 내용을 그대로 반영하는 방식 중 선택 가능해야 함.

### STEP 2 상세 — 평가요소 확인·수정
- 성취기준에서 자동으로 뽑아낸 평가요소 후보를 제시.
- "지식이해 + 과정기능 + ~하기" 형태 문구로 다듬음.
- 이번 문항에서 실제로 평가할 요소만 체크하여 선택.

## 4. 출력 형식 규칙

- 단위: 반(학급) 단위로 작성.
- 학생 식별: 이름 미표시, 학번만 표시.
- 서술 방식: 반별 전체 내용을 한 장으로 통합, 부드러운 문장으로 서술.
- 문체: 생활기록부 규정에 맞춰 "~함", "~임" 등으로 종결.

## 5. 배포 계획

- 전체 내용 정리가 완료되면 GitHub을 통해 배포 예정.

## 6. 미확정 항목

- 10번 항목: 아직 내용 미기재 (추가 요구사항 있을 시 반영 예정).
- 실제 평가계획서 원본 파일: 아직 업로드되지 않음. 업로드 시 STEP 1~5 파이프라인에 맞춰 실제 데이터 분석 진행 가능.
